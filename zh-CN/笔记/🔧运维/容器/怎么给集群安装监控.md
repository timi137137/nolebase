# 怎么给集群安装监控

起因是因为发现用Gitlab自带的Prometheus探针给Runner做监控，几乎什么有用的信息都看不见。

最后在Rancher上发现其实现在的Kubernetes集群有专门的一套监控Helm包，所以记录一下如何安装的。

---

```bash
# 首先添加官方的仓库
helm repo add prometheus-community https://prometheus-community.github.io/helm-charts
# 查看一下我们要的包是否存在，免得出错了
helm search repo prometheus-community
# 把包自带的配置文件导出，文件名自己喜欢就行
helm show values prometheus-community/kube-prometheus-stack > prometheus-stack.values.yaml
```

> [!warning] 
> 请注意必须记得修改配置文件，因为默认是集群IP的服务类型，外网是访问不到的！
> 最少最少也得改成 `NodePort`

```bash
# 安装Helm包，部署名建议需要自己记得，后续可能会使用。命名空间建议创建指定一个自己喜欢的，可以跨空间不用担心。
helm install {部署名} prometheus-community/kube-prometheus-stack -n {命名空间} -f prometheus-stack.values.yaml
# 更新Helm包，用这个
helm upgrade {部署名} prometheus-community/kube-prometheus-stack -n {命名空间} -f prometheus-stack-values.yaml
```

---
### 其他常用指令

```bash
# 查询安装的Pod们
kubectl -n {命名空间} get pods -l "release={部署名}"
# 获取Grafana面板admin用户密码
kubectl -n {命名空间} get secrets prometheus-stack-grafana -o jsonpath="{.data.admin-password}" | base64 -d ; echo
# 如果你没改服务类型的话，你可能会需要转发端口访问Grafana面板
export POD_NAME=$(kubectl --namespace default get pod -l "app.kubernetes.io/name=grafana,app.kubernetes.io/instance=prometheus-stack" -oname)
kubectl -n {命名空间} port-forward $POD_NAME 3000:3000 --address 0.0.0.0
```
